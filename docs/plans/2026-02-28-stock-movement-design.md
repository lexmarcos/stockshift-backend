# Stock Movement Module — Design Document

**Date:** 2026-02-28
**Status:** Approved

## Overview

A new `StockMovement` module that serves as a **unified record of all inventory movements**. Movements are **immediate** (no approval workflow) and the system uses **automatic FIFO** to debit from batches. The module focuses on tracking and auditing.

## Decisions

- **No approval workflow** — movements are recorded and executed immediately
- **StockMovement as unified log** — Transfer module continues operational, but each Transfer execution also generates StockMovement records automatically
- **FIFO automatic** — user only informs product and quantity; the system selects the oldest batch first
- **Reports**: Movement extract (filtered by product, warehouse, type, period) and warehouse summary (comparative)

## Main Flow (Manual Output)

1. User informs: `productId`, `quantity`, `type` (e.g. USAGE), `notes` (optional)
2. System fetches batches for the product in the current warehouse, ordered by `createdAt ASC` (FIFO)
3. Debits from batches in sequence until covering the total quantity
4. Creates `StockMovementItem` for each affected batch
5. Creates `InventoryLedger` entries (preserving existing audit trail)
6. Returns the complete `StockMovement`

## Transfer Integration

When a Transfer is executed, `TransferService` calls `StockMovementService` to automatically generate `TRANSFER_OUT` movements (at source warehouse). When validation completes, `TRANSFER_IN` movements are generated at the destination warehouse.

## API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/stock-movements` | Create movement (usage, gift, loss, purchase, adjustment) |
| `GET` | `/api/stock-movements` | List with filters (type, product, period, warehouse) — **Extract** |
| `GET` | `/api/stock-movements/{id}` | Movement detail |
| `GET` | `/api/stock-movements/warehouse-summary` | **Warehouse summary** (comparative) |

## Reports

- **Extract**: Paginated, filterable by `type`, `productId`, `warehouseId`, `dateFrom`, `dateTo`
- **Warehouse Summary**: Aggregates total quantity by movement type per warehouse in a period
